ID: 24973
Title: After executing `add column`, conversion `binary` to `int` is inconsistent with MySQL
Description:
## Bug Report

Please answer these questions before submitting your issue. Thanks!

## About `binary`:

### 1. Minimal reproduce step (Required)

<!-- a step by step guide for reproducing the bug. -->
```
set global tidb_enable_change_column_type = 1;

drop table tbl;
create table tbl(col_214 decimal(30,8));
replace into tbl values (89687.448);
alter table tbl add column col_279 binary(197) collate binary default 'RAWTdm' not null;

select * from tbl;
alter table tbl change column col_279 col_287 int;
select * from tbl;
```

### 2. What did you expect to see? (Required)
```
mysql> select * from tbl;
+----------------+-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| col_214        | col_279                                                                                                                                                                                               |
+----------------+-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| 89687.44800000 | RAWTdm                                                                                                                                                                                                |
+----------------+-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
1 row in set (0.01 sec)

mysql> alter table tbl change column col_279 col_287 int;
ERROR 1366 (HY000): Incorrect integer value: 'RAWTdm\x00\x00\x00\x00\x00\x00\x00\x00\x00\x00\x00\x00\x00\x00\x00\x00\x00\x00\x00\x00\x00\x00\x00\x00\x00\x00\x00\x00\x00\x00\x' for column 'col_287' at row 1

mysql> select * from tbl;
+----------------+-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| col_214        | col_279                                                                                                                                                                                               |
+----------------+-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| 89687.44800000 | RAWTdm                                                                                                                                                                                                |
+----------------+-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
1 row in set (0.01 sec)
```

### 3. What did you see instead (Required)
```
tidb> select * from tbl;
+----------------+-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| col_214        | col_279                                                                                                                                                                                               |
+----------------+-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| 89687.44800000 | RAWTdm                                                                                                                                                                                                |
+----------------+-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
1 row in set (0.00 sec)

tidb> alter table tbl change column col_279 col_287 int;
Query OK, 0 rows affected (2.52 sec)

tidb> select * from tbl;
+----------------+---------+
| col_214        | col_287 |
+----------------+---------+
| 89687.44800000 |    NULL |
+----------------+---------+
1 row in set (0.00 sec)
```
