ID: 43528
Title: SQL execution efficiency is low when paging is turned on
Description:
## Bug Report
After [PR 41120](https://github.com/pingcap/tidb/pull/41120/files) , when sql condition `limit` is less than 128,tidb will disable paging by default. But when the condition is  `limit 10000`, sql execution efficiency is still better paging is turned off than paging is turned on,especially when there are many regions.
Please answer these questions before submitting your issue. Thanks!

### 1. Minimal reproduce step (Required)
```
mysql>  CREATE TABLE `test` (`id` int(11) DEFAULT NULL,`name` varchar(100) DEFAULT NULL, KEY `idx_id` (`id`));
Query OK, 0 rows affected (0.11 sec)
mysql>  insert into test(id,name) values(1,'ssss'),(2,'ssss'),(3,'ssss'),(4,'ssss')...;
mysql>  split table test index idx_id between ("1") and ("4") regions 4;
```
