Title: Different warnings generated by the same expression
Description:
## Bug Report

Please answer these questions before submitting your issue. Thanks!

### 1. Minimal reproduce step (Required)
```sql
DROP TABLE IF EXISTS t0;
CREATE TABLE t0(c0 NUMERIC);
INSERT INTO t0 VALUES(1);
SELECT c0 FROM t0 WHERE 1 | BIN(100000000);
SHOW WARNINGS;
UPDATE t0 SET c0 = 2 WHERE 1 | BIN(100000000);
SHOW WARNINGS;
```
<!-- a step by step guide for reproducing the bug. -->

### 2. What did you expect to see? (Required)
The expression '1 | BIN(100000000)' should be treated the same in SELECT and UPDATE, which means warning messages should be the same.
### 3. What did you see instead (Required)
SELECT throws a truncate warning.
```sql
+---------+------+------------------------------------------------------------------+
| Level   | Code | Message                                                          |
+---------+------+------------------------------------------------------------------+
| Warning | 1292 | Truncated incorrect INTEGER value: '101111101011110000100000000' |
+---------+------+------------------------------------------------------------------+
1 row in set (0.00 sec)
```
UPDATE throws an out of range error.
```sql
+-------+------+---------------------------------------------------------------+
| Level | Code | Message                                                       |
+-------+------+---------------------------------------------------------------+
| Error | 1690 | BIGINT value is out of range in '101111101011110000100000000' |
+-------+------+---------------------------------------------------------------+
1 row in set (0.00 sec)
```
