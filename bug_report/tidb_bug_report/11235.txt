## Bug Report

Please answer these questions before submitting your issue. Thanks!

1. What did you do?
If possible, provide a recipe for reproducing the error.

I'm trying to batch update/delete hundrads of records using PrepareStatement#executeBatch through JDBC. The method will return an array with updated count(`0` or `1`) or response code(`-2` for success or `-3` for failure) for each record. If in MySQL, all record will return updated count or success code, but in TiDB, only first record of batch will be updated, all of the remaining ones will fail. This behavior can be reproduced by different JDBC driver(`org.mariadb.jdbc.Driver` and `com.mysql.jdbc.Driver`) through TiDB v2.0 to v3.0. The difference only occurs during batch update and delete, for batch insert it is the same with MySQL.
