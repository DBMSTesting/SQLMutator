ID: 177
Title: Oceanbase的timediff功能存在bug
Description:
oceanbase使用timediff时，如果数据的小数点超过2位，计算结果为null
如：select timediff("12:34:56.123", "13:33:22.34567");的计算结果就为null

复现过程：
**mysql5.6/5.7：**
mysql> select timediff("12:34:56.123", "13:33:22.34567");
+--------------------------------------------+
| timediff("12:34:56.123", "13:33:22.34567") |
+--------------------------------------------+
| -00:58:26.22267                            |
+--------------------------------------------+
1 row in set (0.00 sec)

mysql> select timediff("13:34:23.01","13:34:23.002");
+----------------------------------------+
| timediff("13:34:23.01","13:34:23.002") |
+----------------------------------------+
| 00:00:00.008                           |
+----------------------------------------+
1 row in set (0.00 sec)

mysql> select timediff("13:34:23.01","13:34:23.02");
+---------------------------------------+
| timediff("13:34:23.01","13:34:23.02") |
+---------------------------------------+
| -00:00:00.01                          |
+---------------------------------------+
1 row in set (0.00 sec)

**oceanbase:**
MySQL [test]> select timediff("12:34:56.123", "13:33:22.34567");
+--------------------------------------------+
| timediff("12:34:56.123", "13:33:22.34567") |
+--------------------------------------------+
| NULL                                       |
+--------------------------------------------+
1 row in set (0.00 sec)

MySQL [test]> select timediff("13:34:23.01","13:34:23.002");
+----------------------------------------+
| timediff("13:34:23.01","13:34:23.002") |
+----------------------------------------+
| NULL                                   |
+----------------------------------------+
1 row in set (0.00 sec)
