ID: 1945
Title: [Bug]: `stddev_pop()` 返回了预期之外的NULL
Description:
**Describe the bug**
master分支版本，对于参数为常数的`stddev_pop()`，返回了预期之外的NULL

**Fast Reproduce Steps(Required)**
先执行`init.sql`建表，随后执行`error.sql`，出现预期之外的结果。可以尝试多次执行`error.sql`来复现错误。如果多次尝试未果，可以尝试重新建立数据库，再次执行`init.sql`建表

[init.sql.txt](https://github.com/oceanbase/oceanbase/files/15174627/init.sql.txt)
[error.sql.txt](https://github.com/oceanbase/oceanbase/files/15174626/error.sql.txt)

**Expected behavior**
对于参数为常数的`stddev_pop()`，结果不应为NULL.

**Actual Behavior**

MySQL结果：
```
mysql> SELECT
    ->   stddev_pop(
    ->     cast(11.11 as double)) as c1
    -> FROM
    ->   t_qch as ref_0
    -> WHERE
    ->   1=1
    -> GROUP BY
    ->   ref_0.c_s96jklr800;
+------+
| c1   |
+------+
|    0 |
|    0 |
|    0 |
+------+
3 rows in set (0.00 sec)
```

OB的单节点版本有错误，多节点版本正常
多节点：
```
0.000000 | 
0.000000 | 
0.000000 | 
```

单节点：
```
`#NULL`  | 
`#NULL`  | 
0.000000 | 
```
